# EchoLink系统技术规格书

## 一、 系统简介

**EchoLink** 是一款基于大语言模型（LLM）的智能记事助手。它改变了传统的繁琐输入模式，通过识别用户自然语言意图，实现信息的**自动分类存储**与**即时语义查询**。

------

## 二、 App 端功能 (Flutter)

### 1. 工作区 (Workspace)

- **动态列表**：展示记事卡片列表。
- **实时更新**：内置 WebSocket 客户端，监听后端推送指令，实现数据无感刷新。
- **核心交互**：
  - **悬浮按钮**：右下角设有“说”按钮。
  - **交互逻辑**：按住录音，松开上传，等待推送列表局部刷新。

### 2. 我的 (Profile)

- **个人中心**：展示静态个人基础信息、头像及 UID。

------

## 三、 后端服务 (Go)

### 1. 核心模块

> **STT 模块**：集成语音转文字服务（如 Whisper），负责音频流到文本的精准还原。
>
> **意图引擎**：调用 LLM（如 Gemini/GPT）分析文本意图：
>
> - **记录类**：提取结构化 JSON 并写入 MySQL。
> - **查询类**：生成 SQL 统计逻辑，并将结果润色后返回。
>
> **通信模块**：基于 WebSocket 协议，负责将后端处理状态实时推送到前端。

### 2. 存储与接口

- **数据库**：MySQL 8.0+。
- **接口规范**：标准 RESTful API（用于基础数据获取） + WebSocket（用于实时交互）。

------

## 四、 核心业务流程

1. **输入阶段**：用户长按说话 Flutter 采集音频并上传。
2. **处理阶段**：后端 STT 转文字 AI 判定意图 执行数据库 `INSERT` 或 `SELECT` 操作。
3. **反馈阶段**：操作完成后，后端经由 WebSocket 发送 `update` 指令 App 局部刷新展示新卡片或 AI 回答。

------

## 五、 数据库表设计 (`records`)

| **字段名**     | **数据类型**  | **约束**                    | **备注说明**                  |
| -------------- | ------------- | --------------------------- | ----------------------------- |
| **id**         | BIGINT        | PRIMARY KEY, AUTO_INCREMENT | 唯一标识符                    |
| **user_id**    | INT           | NOT NULL, DEFAULT 1         | 用户隔离 ID（预留扩展）       |
| **raw_text**   | TEXT          | NOT NULL                    | 原始语音文本（用于溯源修正）  |
| **category**   | ENUM          | 'bill', 'todo', 'memo'      | AI 识别分类：账单、待办、随笔 |
| **content**    | VARCHAR(255)  | NOT NULL                    | AI 提炼的核心事件内容         |
| **val_amount** | DECIMAL(10,2) | DEFAULT 0.00                | 仅在 `bill` 类型时记录金额    |
| **event_time** | DATETIME      | DEFAULT NULL                | 目标时间（如会议、提醒时间）  |
| **created_at** | DATETIME      | DEFAULT CURRENT_TIMESTAMP   | 系统记录入库时间              |
